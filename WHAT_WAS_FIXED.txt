LIBRARY CHANGE:
  motion@^12.23.24  ──>  framer-motion@^10.16.4

  Why? Your code uses Framer Motion API (AnimatePresence, motion.div, etc.)
       motion library has different API, causing export mismatch
       framer-motion is the correct library for your code

TYPE DEFINITIONS:
  @types/react@^19.0.0      → Tells TypeScript what React exports
  @types/react-dom@^19.0.0  → Tells TypeScript about React DOM

  Why? TypeScript needs to know module structure for type checking
       Without these, TypeScript can't verify your React code is correct

IMPORT UPDATES:
  'motion/react'    ──>    'framer-motion'  (5 files updated)

  Why? Different import paths for different libraries
       motion uses 'motion/react', framer-motion uses 'framer-motion'
