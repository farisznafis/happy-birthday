📋 STEP-BY-STEP INSTALLATION:
═════════════════════════════════════════════════════════════════════════════

Step 1: Install Missing Type Definitions
─────────────────────────────────────────────────────────────────────────────
Run this command in your terminal/command prompt:

  npm install --save-dev @types/react @types/react-dom

This installs TypeScript type definitions for React. You only need to do this once.


Step 2: Install All Dependencies
─────────────────────────────────────────────────────────────────────────────
Run:

  npm install

This downloads all required packages (only needed once).


Step 3: Verify Installation
─────────────────────────────────────────────────────────────────────────────
Run:

  npm run lint

You should see: (no errors output)


Step 4: Start Development Server
─────────────────────────────────────────────────────────────────────────────
Run:

  npm run dev

You should see:

  VITE v6.x.x  ready in XXX ms

    ➜  Local:   http://localhost:3000/
    ➜  Network: http://YOUR_IP:3000/

Note: Keep this terminal/command prompt window open while developing.


Step 5: Open in Browser
─────────────────────────────────────────────────────────────────────────────
Visit: http://localhost:3000

You should now see the beautiful birthday app! 🎉

🎵 HOW TO CHANGE THE MUSIC:
═════════════════════════════════════════════════════════════════════════════

METHOD 1: Use a Different Online URL (EASIEST)
─────────────────────────────────────────────────────────────────────────────

1. Open: src/App.tsx
2. Find this line (around line 237):

   src="https://www.soundhelix.com/examples/mp3/SoundHelix-Song-1.mp3"

3. Replace the URL with your music URL, for example:

   src="https://your-music-site.com/happy-birthday.mp3"

4. Save the file (browser will auto-refresh)
5. Click the volume icon to play your music!


METHOD 2: Use a Local Music File
─────────────────────────────────────────────────────────────────────────────

Step 1: Create a "public" folder (if it doesn't exist) in the project root:
  happy-birthday/
  ├── public/          ← Create this folder
  │   └── music.mp3    ← Add your music file here
  ├── src/
  └── package.json

Step 2: Place your music file (music.mp3) in the public folder

Step 3: Open src/App.tsx and change the audio src to:

   src="/music.mp3"

Step 4: Save and your music will play!


METHOD 3: Find Free Music Online
─────────────────────────────────────────────────────────────────────────────

Visit these sites for free copyright-free music:

  • Pixabay Music: https://pixabay.com/music/
  • FreePik: https://www.freepik.com/
  • YouTube Audio Library: https://www.youtube.com/audiolibrary
  • Bensound: https://www.bensound.com/

Find a song, download it as MP3, and use METHOD 1 or METHOD 2 above.

🐛 TROUBLESHOOTING:
═════════════════════════════════════════════════════════════════════════════

Problem: Blank White Page
─────────────────────────────────────────────────────────────────────────────
Solution:
  1. Open browser DevTools (Press F12)
  2. Click "Console" tab
  3. Look for red errors and copy them
  4. Hard refresh: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac)
  5. Restart dev server: Stop it (Ctrl+C), then run: npm run dev


Problem: TypeScript Errors in Editor
─────────────────────────────────────────────────────────────────────────────
Solution (VSCode):
  1. Command Palette: Ctrl+Shift+P
  2. Type: TypeScript: Restart TS Server
  3. Press Enter

  Or restart VSCode completely.


Problem: "npm: command not found"
─────────────────────────────────────────────────────────────────────────────
Solution:
  1. Install Node.js from: https://nodejs.org/
  2. Download the LTS (Long Term Support) version
  3. Install it
  4. Restart your terminal/command prompt
  5. Try npm install again


Problem: Music Not Playing
─────────────────────────────────────────────────────────────────────────────
Solution:
  1. Check browser console (F12) for errors
  2. Verify URL is correct (test in new browser tab)
  3. Make sure URL is HTTPS (not HTTP)
  4. Check browser volume isn't muted
  5. Check music file size (keep under 5MB)


Problem: Images Not Loading
─────────────────────────────────────────────────────────────────────────────
Solution:
  1. Image URL must be HTTPS
  2. Open image URL in new tab to verify it works
  3. Some image servers may block external access
  4. Try using a different image hosting service
